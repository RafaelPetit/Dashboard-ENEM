PROBLEMAS CORRIGIDOS:
====================

1. ERRO: "Operação 'desvio' não suportada"
   CAUSA: O código usava 'desvio' como operação estatística, mas o calculador só reconhecia 'std'
   CORREÇÃO:
   - Adicionada 'desvio' como operação suportada em STATS_CONFIG.SUPPORTED_OPERATIONS
   - Criado alias 'desvio': self._safe_std no mapeamento de operações
   - Mantida retrocompatibilidade com código existente
   ARQUIVOS MODIFICADOS:
   - data/config.py: Adicionada 'desvio' na lista de operações suportadas
   - data/statistics.py: Criado alias 'desvio' para operação 'std'

2. ERRO: "Erro ao aplicar filtro: 'Dict' nodes are not implemented"
   CAUSA: Uso de pandas.query() com expressões complexas que causavam problemas de parsing
   CORREÇÃO:
   - Implementado método _aplicar_filtros_manual() como fallback
   - Adicionada proteção try/catch para usar filtros manuais quando query() falha
   - Conversão adequada de tipos para filtros de raça/cor (int)
   ARQUIVOS MODIFICADOS:
   - utils/prepara_dados/desempenho/processors.py: Método _aplicar_filtros() atualizado

3. WARNING: "invalid value encountered in scalar divide"
   CAUSA: Divisões por zero ou valores inválidos em cálculos estatísticos
   CORREÇÃO:
   - Adicionadas verificações np.isfinite() em todas as divisões
   - Implementados métodos de cálculo seguros (_safe_mean, _safe_std, etc.)
   - Proteção contra overflow com np.errstate() e np.clip()
   ARQUIVOS MODIFICADOS:
   - utils/expander/expander_desempenho.py: Proteção em cálculos de coeficiente de variação
   - utils/explicacao/explicacao_geral.py: Proteção em divisões
   - utils/visualizacao/graficos_geral.py: Proteção em cálculos de range
   - data/statistics.py: Métodos seguros para todos os cálculos

4. WARNINGS: "overflow encountered in scalar power/reduce"
   CAUSA: Valores extremos causando overflow em cálculos de curtose e assimetria
   CORREÇÃO:
   - Implementada limitação de valores com np.clip(-1e10, 1e10)
   - Adicionado np.errstate(overflow='ignore', invalid='ignore')
   - Verificação de dados suficientes antes de cálculos complexos
   - Fallback para 0.0 em caso de valores não finitos
   ARQUIVOS MODIFICADOS:
   - data/statistics.py: Métodos _calculate_kurtosis() e _calculate_skewness() protegidos
